# Annotation

## 1. 어노테이션이란?

어노테이션(Annotation)은 코드에 추가 정보를 붙이는 문법이다.  
자바에서 클래스, 메서드, 필드, 파라미터 등에 붙여서  
이 코드가 어떤 역할을 하는지, 프레임워크가 어떻게 동작해야 하는지를 알려준다.

즉, 어노테이션은 "이 코드에 대한 설명표" 같은 역할을 한다.

---
## 2. 왜 사용하는가?
   스프링 같은 프레임워크가 어노테이션을 보고 자동으로 동작할 수 있다.
    예를 들면 이 클래스가 컨트롤러인지, 이 메서드는 post 요청을 처리하는지
    이 값은 요청 본문에서 받아와야하는지
---
## 3. 스프링에서 어노테이션이 중요한 이유

스프링은 어노테이션을 보고 객체를 스프링 빈으로 등록하거나,  
HTTP 요청을 특정 메서드에 연결하거나,  
의존성을 주입하는 등의 작업을 수행한다.

- `@Controller`, `@Service`, `@Repository`는 컴포넌트 스캔으로 스프링 빈 등록과 연결된다.
- 생성자에 `@Autowired`가 있으면 스프링이 필요한 객체를 주입한다.
## 4. 지금 프로젝트에서 사용한 주요 어노테이션

### 4-1. `@RestController`
이 클래스가 REST API 요청을 처리하는 컨트롤러임을 나타낸다.  
반환값은 주로 JSON 형태로 응답된다.

```java
@RestController
public class MemberController {
}
```
4-2. ```@RequestMapping("/members")```

컨트롤러의 기본 URL 경로를 설정한다.
이 컨트롤러 안의 메서드는 /members를 공통 경로로 사용한다.

4-3. ```@PostMapping("/signup")```

POST 방식의 /signup 요청을 처리하는 메서드임을 나타낸다.

4-4.``` @RequestBody ```

클라이언트가 보낸 요청 본문(JSON)을 자바 객체로 변환해서 받는다.
